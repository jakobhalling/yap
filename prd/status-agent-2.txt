DECISIONS MADE
--------------
1. AssemblyAIServiceImpl accepts an optional `channelFactory` parameter for dependency
   injection in tests, avoiding the need for a full mock WebSocket server.

2. The service uses `web_socket_channel` package's `ready` future to detect connection
   failures early during startSession().

3. endSession() has a 5-second timeout waiting for SessionTerminated — if it times out,
   it closes the connection anyway and returns whatever finals were collected.

4. Empty transcript segments (empty text) are silently ignored and not emitted.

5. On unexpected WebSocket closure, an error is emitted on the transcript stream (not
   thrown) so the RecordingService can transition to error state and preserve transcript.

6. The RecordingService clears partial text when a final arrives, per the PRD's
   "keep it simple" guidance.

7. Providers use @riverpod annotation (code generation). The .g.dart part files
   will need `dart run build_runner build` to generate.
